# List and Query Task Files

List, filter, sort, and summarize task files generated by the create command. Scans both top-level and category subdirectories.

## Usage

```
/projectask:list
/projectask:list --status todo
/projectask:list --status in-progress
/projectask:list --latest 5
/projectask:list --tag feature
/projectask:list --priority high
/projectask:list --category bugfix
/projectask:list --dir path/to/tasks
/projectask:list --summary
```

**Input:** $ARGUMENTS

---

## Step 1: Parse Arguments

Parse `$ARGUMENTS` for these optional flags. All flags are optional and can be combined:

| Flag | Values | Default | Description |
|------|--------|---------|-------------|
| `--status` or `-s` | `todo`, `in-progress`, `blocked`, `done`, `cancelled` | (all) | Filter by status |
| `--priority` or `-p` | `critical`, `high`, `medium`, `low` | (all) | Filter by priority |
| `--category` or `-c` | any category name | (all) | Filter by category (matches `category` frontmatter field or subdirectory name) |
| `--tag` or `-t` | any string | (all) | Filter by tag (matches any tag in the list) |
| `--latest` or `-l` | integer N | (all) | Show only the N most recently created tasks |
| `--sort` | `created`, `priority`, `status`, `due`, `category` | `created` | Sort field |
| `--order` | `asc`, `desc` | `desc` | Sort direction |
| `--dir` or `-d` | directory path | `.projectasks/` | Base directory to scan |
| `--summary` | (flag, no value) | false | Show a high-level summary instead of individual tasks |

If `$ARGUMENTS` is empty, list all tasks in `.projectasks/` (including subdirectories) sorted by creation date descending.

## Step 2: Scan Task Files

1. Resolve the base directory (default: `.projectasks/`)
2. Scan for task files in the base directory **and all immediate subdirectories**:
   - Run: `ls <base-dir>/task*.md <base-dir>/*/task*.md 2>/dev/null`
   - This finds both top-level tasks and tasks organized by category subdirectory
3. If no files found, report "No task files found in `<dir>`" and stop
4. For each file, read the YAML frontmatter to extract metadata fields: `status`, `priority`, `category`, `created`, `started`, `completed`, `due`, `branch`, `tags`
5. Also extract the title from the first `# Task: ...` heading
6. Extract the task number (NNN) from the filename pattern `task<NNN>-<slug>.md`
7. If `category` is not in the frontmatter but the file is in a subdirectory, infer category from the subdirectory name

## Step 3: Filter

Apply filters in order:
1. **Status filter**: If `--status` specified, keep only tasks matching that status
2. **Priority filter**: If `--priority` specified, keep only tasks matching that priority
3. **Category filter**: If `--category` specified, keep only tasks matching that category
4. **Tag filter**: If `--tag` specified, keep only tasks where `tags` list contains the specified tag

## Step 4: Sort

Sort the filtered results:
- `created` (default): By `created` timestamp
- `priority`: By priority level (`critical` > `high` > `medium` > `low`)
- `status`: By status (`in-progress` > `todo` > `blocked` > `done` > `cancelled`)
- `due`: By `due` date (tasks without due dates go last)
- `category`: Alphabetically by category

Apply `--order` direction (default: `desc` for `created`, `asc` for `priority`/`status`/`due`/`category`).

If `--latest N` is specified, take only the first N results after sorting.

## Step 5: Display

### Default View (table)

Display a formatted table with these columns:

| Column | Source | Description |
|--------|--------|-------------|
| `#` | Filename `taskNNN` | Task number extracted from filename (e.g., `001`, `012`) |
| `Status` | Frontmatter `status` | With status icon prefix |
| `Priority` | Frontmatter `priority` | Priority level |
| `Category` | Frontmatter `category` or subdirectory | Category name |
| `Title` | `# Task: ...` heading | Task title |
| `Created` | Frontmatter `created` | Date only (YYYY-MM-DD) |
| `Due` | Frontmatter `due` | Date only, or empty |
| `Tags` | Frontmatter `tags` | Comma-separated |

Example output:

```
  #   | Status      | Priority | Category    | Title                                    | Created    | Due        | Tags
------|-------------|----------|-------------|------------------------------------------|------------|------------|-----------------------------
  001 | in-prog     | high     | feature     | Cloud API Replacement                    | 2026-03-05 |            | feature, mqtt, websocket
  002 | todo        | high     | test        | End-to-End Integration Testing           | 2026-03-05 |            | testing, integration, e2e
  003 | todo        | medium   | refactor    | Merge Redundant Maps                     | 2026-03-12 |            | refactor, naming
  004 | done        | high     | feature     | Native Authentication & Token Generation | 2026-03-05 | 2026-03-08 | backend, authentication
```

After the table, show: `N tasks found (X todo, Y in-progress, Z done) across K categories`
